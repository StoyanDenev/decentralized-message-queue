# FROST-Ed25519 Threshold Scheme — Soundness (FA-Crypto)

**Subject:** the libsodium-free C99 FROST-Ed25519 stack — `src/crypto/ed25519/`
(field/group + RFC 8032 sign/verify) and `src/crypto/frost/` (trusted-dealer +
DKG keygen, two-round threshold signing, PSS share-refresh). This document is the
*construction* soundness companion to the implementation audits recorded in
[C99CryptoStackAudit.md](C99CryptoStackAudit.md) §6 (Ed25519), §7 (FROST), §8
(DKG), §8b (PSS refresh): the audits answer "did we implement the scheme correctly
+ safely"; this answers "is the scheme sound, and what does each in-tree check buy."

It extends [FrostVerifyDelegation.md](FrostVerifyDelegation.md) (which proved only
the single shipped `frost_verify` primitive) to the now-shipped keygen / signing /
DKG. No source is modified by this document.

---

## 1. Assumptions

- **A1 (Ed25519 EUF-CMA).** Ed25519 (RFC 8032) is existentially unforgeable under
  chosen-message attack. Per `Preliminaries.md` §2.1.
- **A2 (DL / DDH on the prime-order subgroup).** The discrete-logarithm problem in
  the order-`L` subgroup generated by the Ed25519 base point `B` is hard
  (`L ≈ 2²⁵²`). Schnorr/FROST soundness reduces to this.
- **A3 (SHA-512 as a random oracle).** The Fiat–Shamir challenges and binding
  factors model `SHA-512` as a RO, the standard FROST analysis assumption
  (Komlo–Goldberg 2020; RFC 9591).
- **A4 (honest dealer OR honest-majority DKG).** §3 keygen is analysed in two
  modes: trusted-dealer (the dealer is honest) and DKG (no single party is
  trusted; soundness holds as long as the VSS + PoP checks are enforced by each
  honest participant).

All four hold exactly as the rest of the proof corpus uses them; this document
introduces no new cryptographic assumption.

---

## 2. The construction (as implemented)

Scalars are `mod L`; points are order-`L` subgroup elements; `B` is the base point;
`PK = [s]B` is the group public key for group secret `s`.

- **Shamir share** of `s`: a degree-`(t-1)` polynomial `f` with `f(0)=s`; party `i`
  holds `sᵢ = f(i)`. Any `t` shares recover `s = Σ_{i∈S} λᵢ(0)·sᵢ` with Lagrange
  weight `λᵢ(0) = ∏_{j∈S, j≠i} x_j/(x_j − x_i)` (`determ_frost_reconstruct`).
- **Two-round sign** over message `m` by a `t`-subset `S` (`determ_frost_sign`):
  each `i∈S` has nonces `(dᵢ, eᵢ)` and commitments `Dᵢ=[dᵢ]B`, `Eᵢ=[eᵢ]B`; the
  binding factor `ρᵢ = H₁(i, m, {(j,D_j,E_j)})`; the group commitment
  `R = Σ_{i∈S}(Dᵢ + [ρᵢ]Eᵢ)`; the challenge `c = H₂(R ‖ PK ‖ m) mod L`; the
  signature share `zᵢ = dᵢ + eᵢ·ρᵢ + λᵢ(0)·sᵢ·c`; the aggregate `z = Σ_{i∈S} zᵢ`;
  the signature is `(R, z)`. `determ_frost_sign` is a *centralized* convenience
  routine that holds every signer's secrets at once; the *distributed* protocol that
  runs in production splits it into `determ_frost_sign_partial` (each signer computes
  only its own `zᵢ` from its own `(sᵢ, dᵢ, eᵢ)` plus the public `{Dⱼ}`, `{Eⱼ}`) and
  `determ_frost_aggregate` (sum the `zᵢ`, recompute the shared `R`). T-1.1 shows the
  two paths are byte-identical.

## 3. Theorems

### T-1 (Aggregate is a valid Ed25519 signature)

*For any honest `t`-subset `S`, `(R, z)` produced by `determ_frost_sign` satisfies
the RFC 8032 verification equation `[z]B = R + [c]·PK` and is therefore accepted by
any Ed25519 verifier under `PK`.*

**Proof.** Let `r = Σ_{i∈S}(dᵢ + ρᵢ·eᵢ)`. Then
`R = Σ(Dᵢ + [ρᵢ]Eᵢ) = Σ([dᵢ]B + [ρᵢ·eᵢ]B) = [r]B` by the group homomorphism
(`[a]B+[b]B=[a+b]B`, `[k]([a]B)=[ka]B` — exercised as `test-frost-c99` §1 and
resting on the Ed25519 group ops the audit §6 found clean). And
`z = Σ zᵢ = Σ(dᵢ + ρᵢ·eᵢ) + c·Σ(λᵢ(0)·sᵢ) = r + c·s`, because `Σ_{i∈S} λᵢ(0)·sᵢ =
f(0) = s` (Lagrange interpolation at 0, T-2). Hence
`[z]B = [r]B + [c·s]B = R + [c]([s]B) = R + [c]·PK`. Since `c = H₂(R‖PK‖m)` is the
exact Ed25519 challenge, `(R, z)` is a valid RFC 8032 signature under `PK`. ∎
*(Witnessed end-to-end: `test-frost-c99` signs with two distinct quorums and both
verify under the C99 verifier AND OpenSSL `EVP_DigestVerify`.)*

### T-1.1 (Distributed decomposition is byte-identical to the centralized sign)

*The distributed two-round API — each signer's `determ_frost_sign_partial` plus
`determ_frost_aggregate` — produces the SAME `(R, z)` as the centralized
`determ_frost_sign` for the same inputs, and hence inherits T-1.*

**Proof.** `determ_frost_sign`, `determ_frost_sign_partial`, and
`determ_frost_aggregate` all derive `{ρᵢ}`, `R`, and `c` through one shared routine
(`frost_binding_and_challenge`) over the *same* public commitment lists `{Dᵢ}`,
`{Eᵢ}`, message `m`, and group key `PK`; so every party computes identical `ρ_pos`,
`R`, and `c`. `determ_frost_sign_partial` at position `pos` returns exactly
`z_pos = d_pos + e_pos·ρ_pos + λ_pos(0)·s_pos·c` — the same summand the centralized
loop computes for `i = pos`. `determ_frost_aggregate` outputs `R` (recomputed
identically) and `z = Σ_{i∈S} z_i`, which is the centralized `z`. Therefore the two
paths emit bit-for-bit equal 64-byte signatures, and T-1's verification equation
holds for the distributed result. The shared `R` recompute is also the property that
defeats selective-abort: any `t`-of-`K` quorum that reaches aggregation derives the
same `R` regardless of which members participated. ∎
*(Witnessed: `test-frost-c99` asserts `memcmp(aggregate, centralized, 64)==0`, then
re-verifies the distributed aggregate under both the C99 verifier and OpenSSL, and
confirms a single tampered partial breaks the signature.)*

### T-2 (Reconstruction correctness + secrecy)

*Any `t` shares reconstruct `f(0)=s`; any `t−1` shares reveal nothing about `s`.*

**Proof.** Correctness: Lagrange interpolation of a degree-`(t-1)` polynomial from
`t` points at `x=0` is exact; `determ_frost_reconstruct` computes `λᵢ(0)` over the
scalar field `mod L` (`sc_mul/sub/invert`; the inversion of `∏(x_j−x_i)` is well-
defined since the singular case is rejected — see §4). Secrecy: `f`'s coefficients
`a₁..a_{t-1}` are uniform; `t−1` evaluations leave `f(0)` uniform over the field
(Shamir's information-theoretic argument). ∎ *(Witnessed: four distinct `t`-subsets
each reconstruct the same secret in `test-frost-c99` §3.)*

### T-3 (Unforgeability of the threshold scheme)

*Under A1–A3, an adversary controlling up to `t−1` signers cannot produce a valid
signature on a message the honest signers did not jointly sign.*

**Proof sketch (reduction to A2/A3).** FROST is a Schnorr threshold scheme; its
EUF-CMA security reduces to the one-more discrete-log problem in the ROM
(Komlo–Goldberg 2020; Bellare–Crites–Komlo–Maller–Tessaro–Zhu 2022 for the tight
bound). The two-nonce structure with the message-and-commitment-bound binding
factor `ρᵢ = H₁(i, m, commitment-list)` is precisely the FROST defence against the
Drijvers/ROS concurrent-signing attack: because `ρᵢ` binds the full commitment set
and the message, an adversary cannot recombine commitments across concurrent
sessions to cancel the honest contribution. With `≤ t−1` shares the adversary
lacks a `t`-th evaluation and the Lagrange combination cannot yield `s` (T-2
secrecy). This document inherits the literature reduction; the in-tree
contribution is that the implemented `ρᵢ`/`R`/`c`/`zᵢ` match the analysed scheme
(T-1) and that `H₁`/`H₂` are domain-separated (the binding hash carries a
`"DETERM-FROST"` … context, the challenge is the bare Ed25519 hash). ∎

### T-4 (DKG — no single party learns the group secret; cheating dealer caught)

*In the Pedersen/Feldman DKG (`determ_frost_dkg_*`), the group secret
`s = Σᵢ a_{i,0}` is never assembled by any single party, and an honest participant
accepts a dealt share `s_{i,j}` only if it lies on the committed polynomial.*

**Proof.** Each party `i` commits to `Cᵢ,ₖ = [a_{i,k}]B` and proves possession of
`a_{i,0}` with a Schnorr PoP (T-5). Party `j`'s long-term share is
`s_j = Σᵢ f_i(j)`, a `(t,n)` Shamir share of `s = Σᵢ a_{i,0} = Σᵢ f_i(0)` (sum of
`n` degree-`(t-1)` polynomials is degree-`(t-1)` with constant term `s`); `PK =
Σᵢ Cᵢ,₀ = [s]B`. No party ever holds more than its own `a_{i,0}` and the shares
sent to it, so `s` is reconstructable only by a `t`-quorum acting together — never
by one party (under A4's honest-participant assumption). **VSS binding:**
`determ_frost_dkg_verify_share` accepts `s` iff `[s]B = Σ_{k} j^k·Cᵢ,ₖ`. Since
`Σ_k j^k·Cᵢ,ₖ = [Σ_k a_{i,k} j^k]B = [f_i(j)]B`, the check passes iff `s ≡ f_i(j)
(mod L)`; the only `s` satisfying `[s]B = X` for fixed `X` is `dlog(X)`, so a dealer
who sends `s' ≠ f_i(j)` is caught unless it computes a discrete log (A2). ∎
*(Witnessed: `test-frost-c99` §5 — every dealt share passes VSS, a tampered share
is rejected, `[Σ a_{i,0}]B = Σ Cᵢ,₀`, and the long-term shares reconstruct `s` and
sign a valid Ed25519 sig under the DKG `PK`.)*

### T-5 (Proof-of-possession soundness)

*The DKG PoP is a sound Fiat–Shamir Schnorr proof of knowledge of
`a_{i,0} = dlog(Cᵢ,₀)`, defeating rogue-key attacks.*

**Proof.** PoP = `(R, z)` with `R=[k]B`, `c=H(ctx‖0x02‖idx‖Cᵢ,₀‖R)`, `z=c·a_{i,0}+k`;
`determ_frost_dkg_verify_pop` checks `[z]B = R + [c]·Cᵢ,₀`. Completeness:
`[z]B=[c·a_{i,0}+k]B=[c]Cᵢ,₀+R`. Soundness (knowledge extraction): in the ROM, a
prover producing valid `(R,z)` for two distinct challenges `c≠c'` over the same `R`
yields `a_{i,0}=(z−z')/(c−c')` (forking lemma) — so producing a PoP without
`a_{i,0}` reduces to A2. The challenge binds `Cᵢ,₀` (the statement) and `idx`, so a
PoP for `(Cᵢ,₀, idx)` cannot be replayed for a different participant; this is the
rogue-key defence. The deterministic nonce `k = H(ctx‖0x01‖idx‖a_{i,0})` is benign:
the "message" `(Cᵢ,₀, idx)` is itself a function of `a_{i,0}`, so there is never a
second distinct challenge over the same `R` for the same secret — the classic
nonce-reuse leak does not arise. ∎

### T-6 (PSS refresh preserves the group key; rotates shares; cross-epoch shares are useless)

*The Proactive Secret Sharing refresh (`determ_frost_pss_*`) produces new shares
`s'_j` that (a) reconstruct the SAME secret `s` and verify under the SAME public
key `PK=[s]B`, (b) differ from the old `s_j`, and (c) cannot be combined with
pre-refresh shares to recover `s` — provided each participant's zero-hole proof
passes.*

**Proof.** Each participant `i` deals a degree-`(t-1)` polynomial `δ_i` with
`δ_i(0)=0`, committed as `Cᵢ,ₖ=[δ_{i,k}]B`. `determ_frost_pss_verify_commit`
accepts `i` only if `Cᵢ,₀ = [0]B` (the identity), i.e. `δ_i(0)=0` — so no single
participant can inject a non-zero hole. Let `Δ = Σᵢ δ_i`; it is degree `≤ t-1`
with `Δ(0) = Σᵢ δ_i(0) = 0`. Participant `j`'s refreshed share is
`s'_j = s_j + Σᵢ δ_i(j) = f(j) + Δ(j) = (f+Δ)(j)`, a point on the degree-`(t-1)`
polynomial `g := f+Δ`. **(a) Preservation:** `g(0) = f(0)+Δ(0) = s+0 = s`, so any
`t` of the `s'_j` Lagrange-interpolate to `s` (T-2 correctness applied to `g`), and
`[g(0)]B = [s]B = PK` is unchanged. **(b) Rotation:** `s'_j − s_j = Δ(j)`; since
`Δ` is a non-zero polynomial (its coefficients `δ_{·,k≥1}` are random, not all
zero w.h.p.), `Δ(j) ≠ 0` for all but `≤ t-1` of the `j`, so the share *values*
move even though the secret does not. **(c) Cross-epoch uselessness:** old shares
lie on `f`, new shares on `g = f+Δ`; for `t` points to interpolate to `s` they
must lie on a *single* degree-`(t-1)` polynomial through `(0,s)`. A mixed set of
`a` old + `b=t−a` new points (`1≤a≤t−1`) lies on one such polynomial only if
`Δ(j)=0` at all `b` new abscissae, a measure-zero coincidence over the random `Δ`
(probability `≤ (t-1)/2^{252}` of accidental agreement); generically the mixed
interpolation yields `s'' ≠ s`. Thus a mobile adversary holding `≤ t-1` shares
from epoch `e` gains nothing in epoch `e+1`: it must capture `t` shares *within one
epoch*. **VSS during refresh:** each `δ_i(j)` is validated by the SAME Feldman
check (`determ_frost_dkg_verify_share`), which sees `Cᵢ,₀=identity` transparently
(`Σ_k j^k Cᵢ,ₖ = [Σ_k δ_{i,k} j^k]B = [δ_i(j)]B`), so a cheating refresher is
caught modulo A2 exactly as in T-4. ∎
*(Witnessed: `test-frost-c99` §6 — zero-hole commit/verify, the non-zero-hole +
non-identity-`C₀` rejections, per-share VSS, shares-actually-changed, two refreshed
subsets reconstructing the original `s`, the old+new mix FAILING to recover `s`,
and a refreshed quorum signing a valid Ed25519 sig under the unchanged `PK`.)*

## 4. Robustness / canonicality gates (what the hardening buys)

These are not soundness of *knowledge* (T-1/T-3/T-5 hold regardless) but
*uniqueness/non-malleability* of the wire bytes — relevant the moment a signature
or PoP is treated as an identifier (dedup, equivocation evidence, a txid-like
hash):

- **Singular signer-set guard** (`determ_frost_sign`, audit §7.1): a repeated
  x-coordinate makes the Lagrange denominator `∏(x_j−x_i)=0`; `sc_invert(0)=0`
  would collapse `λᵢ` to 0 and silently produce a wrong signature. The `[1,255]` +
  pairwise-distinct guard turns this into a clean `−1`.
- **Signature non-malleability** (Ed25519 verify, audit §6.1): rejecting `S ≥ L`
  (`sc_lt_L`, RFC 8032 §5.1.7) means `(R, S)` has no second form `(R, S+L)`. A FROST
  aggregate inherits this when verified through `determ_ed25519_verify`.
- **Canonical encodings** (audit §6.2 / §8.2): rejecting non-canonical `y ≥ q`
  point encodings (`point_y_is_canonical`, RFC 8032 §5.1.3) and non-canonical PoP
  scalars (`determ_frost_dkg_verify_pop` now applies both gates) preserves "one
  point / one PoP = one encoding."

## 5. Scope and non-goals

- **Implementation correctness + constant-time** are established separately by the
  adversarial audits (C99CryptoStackAudit §1–§8b, incl. §8b PSS refresh — 0
  confirmed findings) and the byte-equal-vs-OpenSSL + KAT cross-validation in
  `test-ed25519-c99` / `test-frost-c99`; this document assumes them and argues the
  construction.
- **RFC 9591 byte-exact interop is NOT claimed.** The binding-factor encoding is
  self-consistent (signer ⇄ aggregator agree; the aggregate verifies as a standard
  Ed25519 signature) but is not yet byte-identical to the FROST(Ed25519,SHA-512)
  ciphersuite vectors. Interop with external FROST tooling is a tracked follow-up;
  it does not affect T-1..T-5 for Determ's own committee, which controls both
  signer and aggregator.
- **Trusted dealer vs DKG.** The trusted-dealer keygen (T-2) is a development /
  testing convenience; the **DKG (T-4/T-5) is the production keygen** for a
  decentralized committee, since a trusted dealer would know `s`. **PSS refresh
  (T-6)** is the epoch-boundary maintenance operation: it re-randomizes the
  share set under the fixed DKG group key, so committee membership can roll over
  and long-lived shares can be retired without a fresh DKG or any change to `PK`.
- **Consensus wiring is out of scope.** Replacing the v1 commit-reveal block
  randomness with the FROST aggregate behind `v2_10_active_from_height`
  (`compute_block_rand`) is the remaining integration step; its safety composes
  with the existing FA3 selective-abort and FA1 K-of-K results once landed.
