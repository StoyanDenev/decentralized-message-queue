# FROST-Ed25519 Threshold Scheme вҖ” Soundness (FA-Crypto)

**Subject:** the libsodium-free C99 FROST-Ed25519 stack вҖ” `src/crypto/ed25519/`
(field/group + RFC 8032 sign/verify) and `src/crypto/frost/` (trusted-dealer +
DKG keygen, two-round threshold signing, PSS share-refresh). This document is the
*construction* soundness companion to the implementation audits recorded in
[C99CryptoStackAudit.md](C99CryptoStackAudit.md) В§6 (Ed25519), В§7 (FROST), В§8
(DKG), В§8b (PSS refresh): the audits answer "did we implement the scheme correctly
+ safely"; this answers "is the scheme sound, and what does each in-tree check buy."

It extends [FrostVerifyDelegation.md](FrostVerifyDelegation.md) (which proved only
the single shipped `frost_verify` primitive) to the now-shipped keygen / signing /
DKG. No source is modified by this document.

---

## 1. Assumptions

- **A1 (Ed25519 EUF-CMA).** Ed25519 (RFC 8032) is existentially unforgeable under
  chosen-message attack. Per `Preliminaries.md` В§2.1.
- **A2 (DL / DDH on the prime-order subgroup).** The discrete-logarithm problem in
  the order-`L` subgroup generated by the Ed25519 base point `B` is hard
  (`L вүҲ 2ВІвҒөВІ`). Schnorr/FROST soundness reduces to this.
- **A3 (SHA-512 as a random oracle).** The FiatвҖ“Shamir challenges and binding
  factors model `SHA-512` as a RO, the standard FROST analysis assumption
  (KomloвҖ“Goldberg 2020; RFC 9591).
- **A4 (honest dealer OR honest-majority DKG).** В§3 keygen is analysed in two
  modes: trusted-dealer (the dealer is honest) and DKG (no single party is
  trusted; soundness holds as long as the VSS + PoP checks are enforced by each
  honest participant).

All four hold exactly as the rest of the proof corpus uses them; this document
introduces no new cryptographic assumption.

## 2. The construction (as implemented)

Scalars are `mod L`; points are order-`L` subgroup elements; `B` is the base point;
`PK = [s]B` is the group public key for group secret `s`.

- **Shamir share** of `s`: a degree-`(t-1)` polynomial `f` with `f(0)=s`; party `i`
  holds `sбөў = f(i)`. Any `t` shares recover `s = ОЈ_{iвҲҲS} О»бөў(0)В·sбөў` with Lagrange
  weight `О»бөў(0) = вҲҸ_{jвҲҲS, jвү i} x_j/(x_j вҲ’ x_i)` (`determ_frost_reconstruct`).
- **Two-round sign** over message `m` by a `t`-subset `S` (`determ_frost_sign`):
  each `iвҲҲS` has nonces `(dбөў, eбөў)` and commitments `Dбөў=[dбөў]B`, `Eбөў=[eбөў]B`; the
  binding factor `ПҒбөў = HвӮҒ(i, m, {(j,D_j,E_j)})`; the group commitment
  `R = ОЈ_{iвҲҲS}(Dбөў + [ПҒбөў]Eбөў)`; the challenge `c = HвӮӮ(R вҖ– PK вҖ– m) mod L`; the
  signature share `zбөў = dбөў + eбөўВ·ПҒбөў + О»бөў(0)В·sбөўВ·c`; the aggregate `z = ОЈ_{iвҲҲS} zбөў`;
  the signature is `(R, z)`. `determ_frost_sign` is a *centralized* convenience
  routine that holds every signer's secrets at once; the *distributed* protocol that
  runs in production splits it into `determ_frost_sign_partial` (each signer computes
  only its own `zбөў` from its own `(sбөў, dбөў, eбөў)` plus the public `{Dвұј}`, `{Eвұј}`) and
  `determ_frost_aggregate` (sum the `zбөў`, recompute the shared `R`). T-1.1 shows the
  two paths are byte-identical.

## 3. Theorems

### T-1 (Aggregate is a valid Ed25519 signature)

*For any honest `t`-subset `S`, `(R, z)` produced by `determ_frost_sign` satisfies
the RFC 8032 verification equation `[z]B = R + [c]В·PK` and is therefore accepted by
any Ed25519 verifier under `PK`.*

**Proof.** Let `r = ОЈ_{iвҲҲS}(dбөў + ПҒбөўВ·eбөў)`. Then
`R = ОЈ(Dбөў + [ПҒбөў]Eбөў) = ОЈ([dбөў]B + [ПҒбөўВ·eбөў]B) = [r]B` by the group homomorphism
(`[a]B+[b]B=[a+b]B`, `[k]([a]B)=[ka]B` вҖ” exercised as `test-frost-c99` В§1 and
resting on the Ed25519 group ops the audit В§6 found clean). And
`z = ОЈ zбөў = ОЈ(dбөў + ПҒбөўВ·eбөў) + cВ·ОЈ(О»бөў(0)В·sбөў) = r + cВ·s`, because `ОЈ_{iвҲҲS} О»бөў(0)В·sбөў =
f(0) = s` (Lagrange interpolation at 0, T-2). Hence
`[z]B = [r]B + [cВ·s]B = R + [c]([s]B) = R + [c]В·PK`. Since `c = HвӮӮ(RвҖ–PKвҖ–m)` is the
exact Ed25519 challenge, `(R, z)` is a valid RFC 8032 signature under `PK`. вҲҺ
*(Witnessed end-to-end: `test-frost-c99` signs with two distinct quorums and both
verify under the C99 verifier AND OpenSSL `EVP_DigestVerify`.)*

### T-1.1 (Distributed decomposition is byte-identical to the centralized sign)

*The distributed two-round API вҖ” each signer's `determ_frost_sign_partial` plus
`determ_frost_aggregate` вҖ” produces the SAME `(R, z)` as the centralized
`determ_frost_sign` for the same inputs, and hence inherits T-1.*

**Proof.** `determ_frost_sign`, `determ_frost_sign_partial`, and
`determ_frost_aggregate` all derive `{ПҒбөў}`, `R`, and `c` through one shared routine
(`frost_binding_and_challenge`) over the *same* public commitment lists `{Dбөў}`,
`{Eбөў}`, message `m`, and group key `PK`; so every party computes identical `ПҒ_pos`,
`R`, and `c`. `determ_frost_sign_partial` at position `pos` returns exactly
`z_pos = d_pos + e_posВ·ПҒ_pos + О»_pos(0)В·s_posВ·c` вҖ” the same summand the centralized
loop computes for `i = pos`. `determ_frost_aggregate` outputs `R` (recomputed
identically) and `z = ОЈ_{iвҲҲS} z_i`, which is the centralized `z`. Therefore the two
paths emit bit-for-bit equal 64-byte signatures, and T-1's verification equation
holds for the distributed result. The shared `R` recompute is also the property that
defeats selective-abort: any `t`-of-`K` quorum that reaches aggregation derives the
same `R` regardless of which members participated. вҲҺ
*(Witnessed: `test-frost-c99` asserts `memcmp(aggregate, centralized, 64)==0`, then
re-verifies the distributed aggregate under both the C99 verifier and OpenSSL, and
confirms a single tampered partial breaks the signature.)*

### T-2 (Reconstruction correctness + secrecy)

*Any `t` shares reconstruct `f(0)=s`; any `tвҲ’1` shares reveal nothing about `s`.*

**Proof.** Correctness: Lagrange interpolation of a degree-`(t-1)` polynomial from
`t` points at `x=0` is exact; `determ_frost_reconstruct` computes `О»бөў(0)` over the
scalar field `mod L` (`sc_mul/sub/invert`; the inversion of `вҲҸ(x_jвҲ’x_i)` is well-
defined since the singular case is rejected вҖ” see В§4). Secrecy: `f`'s coefficients
`aвӮҒ..a_{t-1}` are uniform; `tвҲ’1` evaluations leave `f(0)` uniform over the field
(Shamir's information-theoretic argument). вҲҺ *(Witnessed: four distinct `t`-subsets
each reconstruct the same secret in `test-frost-c99` В§3.)*

### T-3 (Unforgeability of the threshold scheme)

*Under A1вҖ“A3, an adversary controlling up to `tвҲ’1` signers cannot produce a valid
signature on a message the honest signers did not jointly sign.*

**Proof sketch (reduction to A2/A3).** FROST is a Schnorr threshold scheme; its
EUF-CMA security reduces to the one-more discrete-log problem in the ROM
(KomloвҖ“Goldberg 2020; BellareвҖ“CritesвҖ“KomloвҖ“MallerвҖ“TessaroвҖ“Zhu 2022 for the tight
bound). The two-nonce structure with the message-and-commitment-bound binding
factor `ПҒбөў = HвӮҒ(i, m, commitment-list)` is precisely the FROST defence against the
Drijvers/ROS concurrent-signing attack: because `ПҒбөў` binds the full commitment set
and the message, an adversary cannot recombine commitments across concurrent
sessions to cancel the honest contribution. With `вүӨ tвҲ’1` shares the adversary
lacks a `t`-th evaluation and the Lagrange combination cannot yield `s` (T-2
secrecy). This document inherits the literature reduction; the in-tree
contribution is that the implemented `ПҒбөў`/`R`/`c`/`zбөў` match the analysed scheme
(T-1) and that `HвӮҒ`/`HвӮӮ` are domain-separated (the binding hash carries a
`"DETERM-FROST"` вҖҰ context, the challenge is the bare Ed25519 hash). вҲҺ

### T-4 (DKG вҖ” no single party learns the group secret; cheating dealer caught)

*In the Pedersen/Feldman DKG (`determ_frost_dkg_*`), the group secret
`s = ОЈбөў a_{i,0}` is never assembled by any single party, and an honest participant
accepts a dealt share `s_{i,j}` only if it lies on the committed polynomial.*

**Proof.** Each party `i` commits to `Cбөў,вӮ– = [a_{i,k}]B` and proves possession of
`a_{i,0}` with a Schnorr PoP (T-5). Party `j`'s long-term share is
`s_j = ОЈбөў f_i(j)`, a `(t,n)` Shamir share of `s = ОЈбөў a_{i,0} = ОЈбөў f_i(0)` (sum of
`n` degree-`(t-1)` polynomials is degree-`(t-1)` with constant term `s`); `PK =
ОЈбөў Cбөў,вӮҖ = [s]B`. No party ever holds more than its own `a_{i,0}` and the shares
sent to it, so `s` is reconstructable only by a `t`-quorum acting together вҖ” never
by one party (under A4's honest-participant assumption). **VSS binding:**
`determ_frost_dkg_verify_share` accepts `s` iff `[s]B = ОЈ_{k} j^kВ·Cбөў,вӮ–`. Since
`ОЈ_k j^kВ·Cбөў,вӮ– = [ОЈ_k a_{i,k} j^k]B = [f_i(j)]B`, the check passes iff `s вүЎ f_i(j)
(mod L)`; the only `s` satisfying `[s]B = X` for fixed `X` is `dlog(X)`, so a dealer
who sends `s' вү  f_i(j)` is caught unless it computes a discrete log (A2). вҲҺ
*(Witnessed: `test-frost-c99` В§5 вҖ” every dealt share passes VSS, a tampered share
is rejected, `[ОЈ a_{i,0}]B = ОЈ Cбөў,вӮҖ`, and the long-term shares reconstruct `s` and
sign a valid Ed25519 sig under the DKG `PK`.)*

### T-5 (Proof-of-possession soundness)

*The DKG PoP is a sound FiatвҖ“Shamir Schnorr proof of knowledge of
`a_{i,0} = dlog(Cбөў,вӮҖ)`, defeating rogue-key attacks.*

**Proof.** PoP = `(R, z)` with `R=[k]B`, `c=H(ctxвҖ–0x02вҖ–idxвҖ–Cбөў,вӮҖвҖ–R)`, `z=cВ·a_{i,0}+k`;
`determ_frost_dkg_verify_pop` checks `[z]B = R + [c]В·Cбөў,вӮҖ`. Completeness:
`[z]B=[cВ·a_{i,0}+k]B=[c]Cбөў,вӮҖ+R`. Soundness (knowledge extraction): in the ROM, a
prover producing valid `(R,z)` for two distinct challenges `cвү c'` over the same `R`
yields `a_{i,0}=(zвҲ’z')/(cвҲ’c')` (forking lemma) вҖ” so producing a PoP without
`a_{i,0}` reduces to A2. The challenge binds `Cбөў,вӮҖ` (the statement) and `idx`, so a
PoP for `(Cбөў,вӮҖ, idx)` cannot be replayed for a different participant; this is the
rogue-key defence. The deterministic nonce `k = H(ctxвҖ–0x01вҖ–idxвҖ–a_{i,0})` is benign:
the "message" `(Cбөў,вӮҖ, idx)` is itself a function of `a_{i,0}`, so there is never a
second distinct challenge over the same `R` for the same secret вҖ” the classic
nonce-reuse leak does not arise. вҲҺ

### T-6 (PSS refresh preserves the group key; rotates shares; cross-epoch shares are useless)

*The Proactive Secret Sharing refresh (`determ_frost_pss_*`) produces new shares
`s'_j` that (a) reconstruct the SAME secret `s` and verify under the SAME public
key `PK=[s]B`, (b) differ from the old `s_j`, and (c) cannot be combined with
pre-refresh shares to recover `s` вҖ” provided each participant's zero-hole proof
passes.*

**Proof.** Each participant `i` deals a degree-`(t-1)` polynomial `Оҙ_i` with
`Оҙ_i(0)=0`, committed as `Cбөў,вӮ–=[Оҙ_{i,k}]B`. `determ_frost_pss_verify_commit`
accepts `i` only if `Cбөў,вӮҖ = [0]B` (the identity), i.e. `Оҙ_i(0)=0` вҖ” so no single
participant can inject a non-zero hole. Let `О” = ОЈбөў Оҙ_i`; it is degree `вүӨ t-1`
with `О”(0) = ОЈбөў Оҙ_i(0) = 0`. Participant `j`'s refreshed share is
`s'_j = s_j + ОЈбөў Оҙ_i(j) = f(j) + О”(j) = (f+О”)(j)`, a point on the degree-`(t-1)`
polynomial `g := f+О”`. **(a) Preservation:** `g(0) = f(0)+О”(0) = s+0 = s`, so any
`t` of the `s'_j` Lagrange-interpolate to `s` (T-2 correctness applied to `g`), and
`[g(0)]B = [s]B = PK` is unchanged. **(b) Rotation:** `s'_j вҲ’ s_j = О”(j)`; since
`О”` is a non-zero polynomial (its coefficients `Оҙ_{В·,kвүҘ1}` are random, not all
zero w.h.p.), `О”(j) вү  0` for all but `вүӨ t-1` of the `j`, so the share *values*
move even though the secret does not. **(c) Cross-epoch uselessness:** old shares
lie on `f`, new shares on `g = f+О”`; for `t` points to interpolate to `s` they
must lie on a *single* degree-`(t-1)` polynomial through `(0,s)`. A mixed set of
`a` old + `b=tвҲ’a` new points (`1вүӨaвүӨtвҲ’1`) lies on one such polynomial only if
`О”(j)=0` at all `b` new abscissae, a measure-zero coincidence over the random `О”`
(probability `вүӨ (t-1)/2^{252}` of accidental agreement); generically the mixed
interpolation yields `s'' вү  s`. Thus a mobile adversary holding `вүӨ t-1` shares
from epoch `e` gains nothing in epoch `e+1`: it must capture `t` shares *within one
epoch*. **VSS during refresh:** each `Оҙ_i(j)` is validated by the SAME Feldman
check (`determ_frost_dkg_verify_share`), which sees `Cбөў,вӮҖ=identity` transparently
(`ОЈ_k j^k Cбөў,вӮ– = [ОЈ_k Оҙ_{i,k} j^k]B = [Оҙ_i(j)]B`), so a cheating refresher is
caught modulo A2 exactly as in T-4. вҲҺ
*(Witnessed: `test-frost-c99` В§6 вҖ” zero-hole commit/verify, the non-zero-hole +
non-identity-`CвӮҖ` rejections, per-share VSS, shares-actually-changed, two refreshed
subsets reconstructing the original `s`, the old+new mix FAILING to recover `s`,
and a refreshed quorum signing a valid Ed25519 sig under the unchanged `PK`.)*

### T-7 (Committee lifecycle composition вҖ” `PK` is a cross-epoch invariant)

*The full committee lifecycle вҖ” DKG keygen `вҶ’` distributed threshold signing `вҶ’`
PSS refresh `вҶ’` continued signing вҖ” composes so that the group public key `PK`
established once by the DKG is an invariant across an unbounded number of refreshes
and signing epochs. A verifier anchored on `PK` need never update it.*

**Proof (chaining the prior theorems).** Let `PKвӮҖ = ОЈбөў Cбөў,вӮҖ = [s]B` be the key the
DKG fixes (T-4), with `s` never assembled anywhere. Index epochs `e = 0,1,2,вҖҰ`,
where each `eвҶ’e+1` boundary is a PSS refresh. By T-6, a refresh maps the share set
on `f_e` to a share set on `f_{e+1} = f_e + О”_e` with `О”_e(0)=0`, so
`f_{e+1}(0) = f_e(0) = s` and `PK_{e+1} = [f_{e+1}(0)]B = PK_e`; by induction
`PK_e = PKвӮҖ` for all `e` (the zero-hole gate `determ_frost_pss_verify_commit` is
what forbids any participant from perturbing the constant term, so the induction
step cannot be broken by one party). Within any epoch `e`, a `t`-of-`n_e` quorum
signs by T-1/T-1.1 and the result is a standard Ed25519 signature under `PK_e = PKвӮҖ`
(T-1), verifiable by any RFC 8032 verifier holding only `PKвӮҖ`. The selective-abort
resistance is per-epoch and self-contained: by T-1.1 every `t`-subset that reaches
aggregation in epoch `e` derives the *same* `R` from the public round-1
commitments, so a withholding minority cannot bias that epoch's output, and this
holds independently in each epoch regardless of refreshes. **Committee handover**
(membership change at a boundary) is the same algebra: PSS with the new index set
re-randomises shares onto `f_{e+1}` while preserving `f_{e+1}(0)=s` вҖ” retiring an
old member's share is exactly the "old shares useless after refresh" property of
T-6(c). Hence a light client or on-chain anchor that pins `PKвӮҖ` once (at genesis or
the first DKG) validates every threshold signature the committee ever produces,
across all refreshes and handovers, with no key update. вҲҺ
*(This is the contract the v2.10 consensus wiring + light-client beacon verification
rely on: one fixed `PK` per chain epoch-group; see `V210ImplementationRoadmap.md`
Phase C/D. Witnessed compositionally вҖ” `test-frost-c99` exercises each link: DKG В§5,
distributed sign В§"4b", PSS-then-sign-under-unchanged-`PK` В§6.)*

## 4. Robustness / canonicality gates (what the hardening buys)

These are not soundness of *knowledge* (T-1/T-3/T-5 hold regardless) but
*uniqueness/non-malleability* of the wire bytes вҖ” relevant the moment a signature
or PoP is treated as an identifier (dedup, equivocation evidence, a txid-like
hash):

- **Singular signer-set guard** (`determ_frost_sign`, audit В§7.1): a repeated
  x-coordinate makes the Lagrange denominator `вҲҸ(x_jвҲ’x_i)=0`; `sc_invert(0)=0`
  would collapse `О»бөў` to 0 and silently produce a wrong signature. The `[1,255]` +
  pairwise-distinct guard turns this into a clean `вҲ’1`.
- **Signature non-malleability** (Ed25519 verify, audit В§6.1): rejecting `S вүҘ L`
  (`sc_lt_L`, RFC 8032 В§5.1.7) means `(R, S)` has no second form `(R, S+L)`. A FROST
  aggregate inherits this when verified through `determ_ed25519_verify`.
- **Canonical encodings** (audit В§6.2 / В§8.2): rejecting non-canonical `y вүҘ q`
  point encodings (`point_y_is_canonical`, RFC 8032 В§5.1.3) and non-canonical PoP
  scalars (`determ_frost_dkg_verify_pop` now applies both gates) preserves "one
  point / one PoP = one encoding."

## 5. Scope and non-goals

- **Implementation correctness + constant-time** are established separately by the
  adversarial audits (C99CryptoStackAudit В§1вҖ“В§8b, incl. В§8b PSS refresh вҖ” 0
  confirmed findings) and the byte-equal-vs-OpenSSL + KAT cross-validation in
  `test-ed25519-c99` / `test-frost-c99`; this document assumes them and argues the
  construction.
- **RFC 9591 byte-exact interop is NOT claimed.** The binding-factor encoding is
  self-consistent (signer вҮ„ aggregator agree; the aggregate verifies as a standard
  Ed25519 signature) but is not yet byte-identical to the FROST(Ed25519,SHA-512)
  ciphersuite vectors. Interop with external FROST tooling is a tracked follow-up;
  it does not affect T-1..T-5 for Determ's own committee, which controls both
  signer and aggregator.
- **Trusted dealer vs DKG.** The trusted-dealer keygen (T-2) is a development /
  testing convenience; the **DKG (T-4/T-5) is the production keygen** for a
  decentralized committee, since a trusted dealer would know `s`. **PSS refresh
  (T-6)** is the epoch-boundary maintenance operation: it re-randomizes the
  share set under the fixed DKG group key, so committee membership can roll over
  and long-lived shares can be retired without a fresh DKG or any change to `PK`.
- **Consensus wiring is out of scope.** Replacing the v1 commit-reveal block
  randomness with the FROST aggregate behind `v2_10_active_from_height`
  (`compute_block_rand`) is the remaining integration step; its safety composes
  with the existing FA3 selective-abort and FA1 K-of-K results once landed.
